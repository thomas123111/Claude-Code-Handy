# Topical Map Generator — Phase Instructions & Reference

## Reference Models: Gold Standard Examples

### Example A: QR Code Generator (SaaS/Tool Context)

**Core Section (BOFU/Monetization)**
- **Brand Identity:** "QR Code Creator" (Brand) vs "Generator" (Synonym)
- **Predicate Clusters:** *Scan* (Android, iPhone), *Generate* (Python, HTML5), *Validate* (Security)
- **Specific Use Cases:** QR for Wifi (→ "Password Sharing"), QR for Payment (→ "Fraud"), Restaurant Menu
- **Competitor Identity:** Beaconstac, QR Code Monkey, Shopify (Define them to control the Knowledge Graph)

**Outer Section (TOFU/MOFU/Context)**
- **The Border:** Starts at "Object Hyperlinking" and "QR Code Alternatives"
- **Comparative Nodes:** Barcode vs QR, NFC, RFID
- **Historical Bridges:** History of Barcodes, Who is Denso Wave? Who is Norman Joseph Woodland?
- **Broad Concepts:** Internet of Things (IoT), Semantic Web, Inventory Management

### Example B: Electric Vehicle (Product/Engineering Context)

**Core Section (BOFU/Monetization)**
- **Component Centrality:** Batteries (Lithium-ion), Charging (Home vs Station), Connectors (CCS, Tesla)
- **Performance Metrics:** Horsepower (linked to Weight), Range (linked to Battery), Torque
- **Differentiation:** Electric Motor Types (Synchronous vs Induction) — distinct from "Engines"
- **Demographics:** EVs for Kids, Teens, Women (based on search behavior)

**Outer Section (TOFU/MOFU/Context)**
- **The Border:** Starts at "History of Electricity" and "Lithium Mining"
- **Scientific Bridges:** How a DC Motor Works, What is Lithium? What is Cobalt?
- **Historical Context:** Who is Karl Benz? History of the Automobile
- **Broader Life Scenarios:** "Moving House" (Utility), "Road Trips" (Range Extenders)

## Phase 1: Strategy, Architecture Definition & General Master Plan

### Instructions

1. **Anchor, Architecture & Dynamic Adaptation**
   - Analyze the Central Entity through the lens of Source Context and Company USP
   - **Dynamic Adaptation:** Explicitly analyze the business model and *reorder* map priority based on it
   - **Industry-Specific Overrides:**
     - *Local SEO:* Enforce nested URL hierarchy. Build "Location Hub Pages" → granular sub-pages (`/location/core-service/sub-service/`)
     - *SaaS/Software:* Create "Versus" and "Alternative" pages to capture competitor traffic
     - *E-commerce:* Use Vertical Specialization (2-3 high-margin categories) OR Horizontal Authority (user problems spanning categories)
   - Reverse engineer competitors, consolidate variations, filter for actual service relevance
   - Categorize keywords: Short-head (Core), Mid-tail (Services), Long-tail (Authority)

2. **General Blueprint & Full Journey Mapping**
   - Outline the comprehensive pillar (hub) page concept
   - Map across the full buyer journey: Awareness (TOFU/Unaware) → Consideration (MOFU/Problem Aware) → Decision (BOFU/Solution Aware) → Post-Purchase (Retention)
   - Plan cross-linking structure between clusters and hub

3. **Semantic Formula & Predicates**
   - Build topics using: *Topic = Knowledge Domain (Noun) + Contextual Domain (Condition)*
   - Group topics by action (Predicate), prioritize Unique → Root → Rare attributes
   - Utilize Hyponyms and Meronyms

4. **Competitor Game Theory & Entity Chain Walking**
   - **"Page 2 & 3" Rule:** Ignore DA giants (government sites, massive directories). Find niche value from specialized competitors
   - Define competitor identities to control the Knowledge Graph
   - Extract attributes by "walking the chain" (Parent/Child/Sibling)
   - Imitate a novice user to find hidden intents

### Phase 1 Output Format

```

  - *Hub Page Concept:* [Brief description]

## Phase 2: Construct the Core Section (Decision & Consideration)

> Do NOT generate a table. Use nested bullet-point structure only.

### Instructions

1. **Definition:** The Core Section covers Transactional/Conversion Intent — Decision and Consideration stages.

2. **Keyword Discovery Methodology (Crucial)**
   - If query fan-outs not provided, **simulate AI query fan-outs**: break complex transactional queries into subqueries
   - **Alphabet Soup Method:** Simulate niche modifiers by running through the alphabet (e.g., "Services for Actors, Builders, Creatives...")
   - Extract entities, identify high-intent modifiers ("Best," "Review," "Cost")
   - Perform gap analysis against competitors

3. **Stage Integration**
   - *Decision Stage:* Comprehensive Product Description, Documentation & Manuals, Pricing & Plans, Case Studies, Compliance content
   - *Consideration Stage:* Reviews & Testimonials, Studies & Research, Comparison & Alternatives, Product Q&As, Community & Forum Content

4. **Three-Level Hierarchy:** Build using L1 Hub → L2 Seed → L3 Node. Follow the "Roundup & Vertical Expansion Protocol" — every item in a roundup gets an individual review node.

5. **Silo Integrity:** Finish every Node for a Seed before moving to the next Seed.

## Phase 3: Construct the Outer Section (Awareness & Post-Purchase)

> Do NOT generate a table. Use nested bullet-point structure only.

### Instructions

1. **Definition:** The Outer Section covers "Know" Intent — Awareness and Post-Purchase stages. It forms the "Outer Border" where the product ends and the concept begins.

2. **Keyword Discovery Methodology**
   - Simulate AI query fan-outs for broad informational topics
   - Reverse-engineer the user's thought process to find emotional triggers, hidden contexts, scientific/historical bridges

3. **Stage Integration**
   - *Awareness Stage:* How-To Guides & Tutorials, Thought Leadership & Expert Advice, FAQs, New product releases
   - *Post-Purchase Stage:* Knowledge base articles, Troubleshooting Guides & FAQs

4. **Scale & Volume Rules**
   - *High-Complexity Niches* (Law, Medical, YMYL): **30-35+ supporting nodes** per core service
   - *Low-Complexity Niches* (Basic Bookkeeping, simple trades): **7-10 supporting nodes** per core service
   - **Batching Rule:** If planning more than 20 items, PAUSE and ask to generate the next batch.

5. **Authority Silos:** Build using Three-Level Hierarchy (L2 Seed → L3 Node). Utilize "Object Hyperlinking" (Glossary Silos) and semantic precision (e.g., separating "Engine" vs "Motor").

```

## Phase 4: Linking Strategy & Coverage Mapping

> This phase is strictly Strategic Planning. No tables.

### Instructions

1. **Analyze & Plan:** Review all nodes from Phases 2 and 3.
2. **Bridge Term Identification:** Identify specific Bridge Terms (nouns) connecting Outer Section to Core Section.
3. **Linking Logic Definition:**
   - Roundup-to-Review links
   - Double internal linking for multi-category tools
   - Inward Flow (Outer → Core)
   - Hub & Spoke Vertical logic
   - Reciprocal Horizontal logic
4. **Coverage Check:** Map every Required Question from the Input Data to a specific Node.

### Phase 4 Output Format

```
## 4. Internal Linking Strategy & Bridge Terms

- **Bridge Term Analysis:** [Specific terms bridging distinct domains]
- **Contextual Link Examples:** [Example sentences linking Outer → Core using Bridge Terms]
- **Vertical & Horizontal Logic:** [Structural flow instructions]
- **Required Question Mapping:** [Each input question → specific Node]
```

## Phase 5: Final Master Map Compilation

> This is the ONLY phase where you output a Markdown Table.

### Instructions

1. **Compile the Table:** Consolidate all bulleted plans from Phases 2 and 3.
2. **URL Logic Constraint:**
   - Use logical, hierarchical nested subfolder structure for topical depth
   - Group related concepts using parent-child relationships (e.g., `/parent-topic/modifier/`)
   - No dates in URLs — slugs must be intent-based

## Global Constraints (Apply Across All Phases)

- **Strict Entity-Attribute Pairing:** Entity + Attribute pair used on ONE single page only. No overlapping variations.
- **No Visual Semantics:** No layout/design instructions. Focus strictly on topical structure and URLs.
- **No Index Blending/Cannibalization:** Don't create URLs for minor keyword variations — merge into Macro-contexts.
- **Funnel Completeness:** All stages must be covered: Awareness, Consideration, Decision, Post-Purchase.
- **Attribute Priority:** Unique → Root → Rare.
- **No Creative Titles:** Raw, long-term keywords only.
- **No Hallucinated Competitor Data:** If competitor sitemaps are NOT provided, do not invent URLs. Rely on training data to simulate the competitive landscape; state when simulating.
